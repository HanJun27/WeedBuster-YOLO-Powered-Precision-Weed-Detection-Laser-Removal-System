## 4. v3.9.9 修了什么（重点）

### Bug A：PID 振荡 / 舵机"疯一般动" 🐛

**根因**：
```
PID_OUTPUT_LIMIT = 5.0°/tick × FSM 10Hz tick = 理论 50°/s
但舵机 + 相机 + BPU 总响应延迟 ~150ms，跟不上
→ PID 用过渡态画面算误差 → 又算大 delta → 转更多 → 失稳
```

**修复**：
- `PID_KP`：0.05 → **0.03**（比例降低 40%）
- `PID_OUTPUT_LIMIT`：5.0° → **1.5°**（单步 ≤ 15 像素移动）
- `PID_KD`：0.02 → **0.025**（增加阻尼）
- 加 `PID_DEADBAND_PX = 6`：误差<6 时 P 项减半（防收敛区震荡）
- 加 `PID_SATURATION_FRAMES = 3`：连续 3 帧输出饱和 → 跳一帧 + 衰减积分

### Bug B：**"目标坐标只用第一次"陷阱** 🐛🐛（隐藏更深）

**根因**（详细物理推导）：
```
舵机转 5° → 视场偏移 5° → 光斑画面位置移动约 50 像素
SPOT_JUMP_MAX_PX = 50 → 触发"帧间稳定性抑制"
→ _detect_spot_now 返回旧的 _last_valid_spot
→ PID 用滞后的 spot 算误差 → 永远算出"目标在原位"
→ 看起来像"只用了第一次坐标"
```

**修复（核心思路）**：当 PID 主动驱动舵机时，**已知视场在变**，所以跳过帧间稳定性检查：

```python
# vision_servo.py _fsm_step:
self._pid_actively_moving = (self.fsm_state in (STATE_COARSE, STATE_PID))

# _detect_spot_now:
if self._pid_actively_moving:
    # 信任新位置，不做 jump 抑制
    self._last_valid_spot = raw_spot
    return raw_spot
# 静止状态（IDLE/LOCKED）才做抑制
```

**辅助修复**：
- `SPOT_JUMP_MAX_PX`：50 → **100**（即使做检查也容忍正常视场移动）
- `SPOT_JUMP_TOLERATE_FRAMES`：3 → **2**（减少抑制延迟）

### 其他小改动

- 视觉伺服画面去掉绿色"红光斑应到位置"十字（战友说调试期辅助，逻辑已验证）
- 右侧面板去掉"红光斑应到"那一行
- 重新统一 yolo_detector.py（BPU 优先 + CPU fallback，替换队友带 .to('npu') bug 的版本）

## 5. IBVS 架构（写 PPT/答辩用）

本系统是经典 IBVS（Image-Based Visual Servoing）在 2-DOF pan-tilt 云台上的工程实现：

```
特征空间   : 图像像素 (cx, cy)
当前特征 s : B = R + Δ（蓝紫预测落点 = 红光斑实测 + 标定二偏移）
期望特征 s*: T（YOLO 杂草中心像素）
误差       : e = s* - s
控制律     : Δyaw   = PIXEL_TO_YAW_DEG  × e_x
            Δpitch = PIXEL_TO_PITCH_DEG × e_y
            （等价于雅可比矩阵对角元素的简化解耦 PID）
驱动量     : 云台 yaw/pitch 增量（2 DOF）
```

**答辩稿一句话**：
> "本系统是 IBVS 在 pan-tilt 云台上的工程化实现，用解耦 PID 替代了经典雅可比伪逆控制律，在保留 IBVS 像素空间收敛保证的同时，避免了雅可比矩阵在线计算与目标深度估计的复杂度。"

| 参数 | 默认 | 含义 | 获取方法 |
|---|---|---|---|
| `PIXEL_TO_YAW_DEG` | -0.10 | 1 像素 → 多少度 yaw | servo_direction_test.py + 实测 |

| Delta_X, Delta_Y | yaml 存 | 蓝紫-红激光偏移 | 跑 calib_laser 用 RGB 标定 |

### 6.2 PID 参数（v3.9.9 新值，浏览器可实时改）

| 参数 | v3.9.9 默认 | 调整方向 |
|---|---|---|
| `PID_KP` | **0.03** | 震荡 → 调小（试 0.02）；响应慢 → 调大（试 0.04） |
| `PID_KI` | 0.001 | 稳态误差大 → 调大；缓慢漂移 → 调小 |
| `PID_KD` | **0.025** | 超调严重 → 调大；响应钝 → 调小 |
| `PID_OUTPUT_LIMIT` | **1.5°** | 振荡 → 调小到 1.0；收敛慢 → 调到 2.0 |
| `PID_DEADBAND_PX` | 6 | 收敛区震荡 → 调大到 8 |
| `PID_TOLERANCE_PX` | 3 | 收敛精度（hypot 距离） |
| `PID_LOCK_FRAMES` | 5 | 连续多少帧 < tolerance 算锁定 |

### 6.3 红光斑检测（v3.9.5 R-max(G,B) 算法）

| 参数 | 默认 | 调整场景 |
|---|---|---|
| `RED_DOMINANCE_MIN` | 30 | 检测不到 → 调小；干扰多 → 调大 |
| `SPOT_ROI_SIZE` | 200 | 光斑跑出 ROI → 调大到 300 |
| `SPOT_JUMP_MAX_PX` | **100** | （v3.9.9 调大）正常情况不用改 |

## 7. YOLO ↔ vision_servo 接口契约

### 7.1 发布格式

**有目标**：
```json
{
  "detected": true,
  "frame_id": 1234,
  "boxes": [
    {"cx": 320, "cy": 240, "w": 45, "h": 60, "confidence": 0.92, "label": "weed"}
  ]
}
```

**无目标（心跳，必须发）**：
```json
{"detected": false, "boxes": []}
```

### 7.2 约束（不能违反）

| 约束 | 违反后果 |
|---|---|
| **10Hz 持续发布**（含心跳） | 数据 > 0.5s 不刷新，PID 回退首帧锁定 |
| **坐标必须 RGB 640×480 像素整数** | 越界或位置错乱 |
| **多目标用 boxes 数组**，伺服自动选离中心最近的 | 不影响功能 |

### 7.3 调试技巧

```bash
# 看 YOLO 发布频率（期望 10±2 Hz）
ros2 topic hz /yolo/weed_detected

# 看 JSON 内容
ros2 topic echo /yolo/weed_detected --once

# 不依赖真 YOLO 模拟一个目标
ros2 topic pub -r 10 /yolo/weed_detected std_msgs/String \
  '{data: "{\"detected\":true,\"cx\":320,\"cy\":240}"}'
```

## 15. 关键工程教训
